# 测试场景描述编写指南

## 基本格式

用一句话描述你的测试需求，包含以下要素：

```
[N个并发用户] [跑多久/循环多少次] [检查什么] [提取什么]
```

## 常用模板

### 模板1：简单压测

```
100 个用户跑 5 分钟，检查状态码 200
```

### 模板2：带预热和断言

```
50 个用户预热 30 秒，持续 10 分钟，响应时间不超过 2 秒，检查返回体包含 success
```

### 模板3：带参数提取

```
200 用户跑 3 分钟，检查返回码 200，提取 token 给后续请求使用
```

### 模板4：数据驱动

```
100 用户从 data.csv 读取用户名密码，跑 1000 次，检查登录成功
```

### 模板5：多接口场景

```
先登录（50并发），获取 token 后查询用户信息（50并发），持续 5 分钟
```

## 参数速查

| 参数 | 说明 | 示例 |
|------|------|------|
| 并发数 | 同时运行的虚拟用户数 | 100、500、1000 |
| 持续时间 | 测试运行时长 | 5分钟、300秒、1小时 |
| 预热时间 | 逐步增加并发到目标值的时间 | 30秒 |
| 循环次数 | 每个用户重复执行的次数 | 100次、永远 |
| 状态码 | 期望的 HTTP 响应状态码 | 200、201 |
| 响应时间 | 期望的最大响应时间 | 2秒、500ms |
| 返回体验证 | 验证响应体包含/等于某内容 | 包含 success |
| JSON 字段验证 | 验证 JSON 响应的某个字段 | token 不为空 |
